파일 시스템 기본 이론

파일 이란?
- Usermode 커널모드로 진입할 수 있는 인터페이스
- 파일마다 고유의 I/O 버퍼가 있다

파일 시스템 = 파일 관리 체계
: 파일 단위의 정보를 생성, 삭제, 저장, 검색할 수 있도록 정보를 구성하는 방법

HDD 안에는 Track 과 sector로 이루어져 있다
Track = 단위
sector = 범위, 512bytes

파일의 논리구조가 '선형'구조 이다 그렇기에 스트리밍이 가능하다

파일스트리밍 + 버퍼 + 큐

파일은 Disk(보조기억장치) 사용하는 방법 이다

메모리의 파일 입출력 지점
입출력 지점 메모리 개방 -> 필요데이터 할당 -> 파일 입출력 지점 변겅 -> 다음 데이터 할당 -> 지점 변경

변수, 메모리 동적 할당 ... : RAM 사용법

텍스트 파일
파일에 저장된 정보가 모두 ASCII 코드나 문자열로 해석할 수 있는 정보

파이너리 파일
텍스트를 제외한 모든 파일
텍스트 와 바이너리의 차이
'-1' 을 EOF(End Of File) 로 인식하면 텍스트
아니라면 바이너리
